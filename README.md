# AutoPilot Framework

Unityゲームに後付けできる**汎用の自動プレイ(ボット)フレームワーク**です。
ポート&アダプタ設計により、コアはゲームを一切参照せず、ゲームごとに薄いアダプタを書くだけで
自動プレイを実現できます。さらに、外部AIエージェントやCLIからの指示を受けて、
ゲームの実行・結果確認・再実行を自律的に回す**自律駆動ループ**も同梱します。

このリポジトリは Unity Package Manager (UPM) パッケージとして、ゲーム固有の依存を持たずに
任意の Unity プロジェクトへ追加できます。

## できること

- ゲームを Play したときに自動でボットを起動する
- 仮想ゲームパッド入力で、実プレイヤーと同等の操作を再現する
- センサーでゲーム状態を観測し、フェーズごとにタスクを切り替える
- 外部から command.json を受けて Play / 停止 / 実行時間制御を行う
- 実行ログとレポートを出力して、外部エージェントが次のアクションを判断できるようにする

## 導入手順

1. Unity プロジェクトにこのパッケージを追加する
2. 使うゲームに合わせてアダプタを実装する
3. Enable On Play もしくは起動フラグで AutoPilot を有効化する
4. Play するとボットが起動し、必要に応じて自律ループが回る

## 自律駆動の流れ

1. 外部から command.json を置く
2. Unity エディタ側のブリッジが command を受け取る
3. Play が開始され、ボットが起動する
4. 指定時間後に停止する
5. status.json / report.json で次の判断材料を残す

この流れが実現できれば、導入は「単なる自動操作」ではなく「自律駆動」へ到達したとみなせます。
